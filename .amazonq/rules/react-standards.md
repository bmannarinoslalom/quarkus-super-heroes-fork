# React/Frontend Standards

## Component Structure
- Use functional components with hooks
- Served via Quarkus Quinoa (no separate Node server)

## Build Configuration
- Environment variables injected at build time via ngx-env plugin
- Runtime config via `env.js` generated by Quarkus server
- Access runtime config via `window.APP_CONFIG`

## API Communication
- Base URL controlled by `API_BASE_URL` environment variable
- Default: `http://localhost:8082`
- Use `CALCULATE_API_BASE_URL=true` to auto-compute from hostname

## File Structure
```
ui-super-heroes/
├── src/
│   └── app/          # React application code
└── deploy/
    ├── docker-compose/
    └── k8s/
```

## Routing Configuration
- Minikube/Kubernetes: Use Ingress with path-based routing
- OpenShift: Use Routes with hostname-based routing
- UI communicates to fights API via `/api/fights` path

## Environment Variables
- `API_BASE_URL`: Full URL to fights service
- `CALCULATE_API_BASE_URL`: Auto-compute from browser URL (OpenShift)
